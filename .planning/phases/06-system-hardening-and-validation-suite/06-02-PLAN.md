Create `scripts/validate-phase6.sh` following the exact pattern from validate-phase1.sh through validate-phase5.sh.

Header: Run ON IPFire. Exits 0 if all pass, 1 if any fail.

Use pass()/fail()/skip() functions with PASS/FAIL/SKIP counters.

**Checks to implement:**

**[HARD-01] Service audit:**
- Check `ss -tlnp` output against known-good baseline ports (per D-04): 53 (unbound), 22 (sshd), 81/444/1013 (httpd)
- Any additional listening TCP port is a finding: fail() with details
- Check Pakfire manifest exists at `/root/firewall-repo/manifests/pakfire-manifest.txt`
  - If exists, diff against expected manifest: `diff /root/firewall-repo/manifests/pakfire-manifest-expected.txt /root/firewall-repo/manifests/pakfire-manifest.txt`
  - If diff shows unexpected packages, fail() with list
  - If manifest file missing, skip() — needs to be generated first

**[HARD-02] File permissions:**
- Check key file permissions using `stat -c "%a" <file>`:
  - `/etc/ssh/sshd_config` should be 600
  - `/etc/sysconfig/firewall.local` should be 700
  - `/root/.ssh/` should be 700
  - `/root/.ssh/authorized_keys` should be 600 (if exists)
  - `/root/integrity-baseline.sha256` should be 600 (if exists)
- pass() each correct, fail() each incorrect with actual vs expected

**[HARD-03] File integrity baseline:**
- Call `bash "${SCRIPT_DIR}/check-integrity.sh" --verify`
- Map exit code: 0=pass(), 1=fail(), 2=skip("integrity mismatch — may need --update-baseline after intentional changes")

**[HARD-04] Kernel hardening (sysctl):**
- Check each hardened parameter via `sysctl -n`:
  - `net.ipv4.conf.all.send_redirects` == 0
  - `net.ipv4.conf.default.send_redirects` == 0
  - `net.ipv4.conf.all.accept_source_route` == 0
  - `net.ipv4.conf.all.accept_redirects` == 0
  - `net.ipv4.conf.all.rp_filter` == 1
  - `net.ipv4.tcp_syncookies` == 1
  - `net.ipv6.conf.all.accept_redirects` == 0
- Also verify `net.ipv4.ip_forward` == 1 (MUST remain 1 for routing — fail if 0)
- pass() each correct, fail() each incorrect with actual value

**[HARD-05] WUI certificate:**
- Check `/etc/httpd/server.crt` exists: pass() or fail()
- Check `/etc/httpd/server-ecdsa.crt` exists: pass() or fail()
- Extract validity dates via `openssl x509 -in /etc/httpd/server.crt -noout -enddate`
- If cert expires within 365 days, fail() with expiry date
- If cert valid > 365 days, pass() with expiry date
- skip() fingerprint match verification — requires docs/wui-certificate.md to be populated first

**Footer:** Print summary: "Phase 6: {PASS} passed, {FAIL} failed, {SKIP} skipped"

<task type="auto">
  <name>Task 2: validate-all.sh — Unified validation orchestrator</name>
  <files>scripts/validate-all.sh</files>
  <action>
Create `scripts/validate-all.sh` as the unified acceptance test orchestrator (per D-11, D-12, D-13, D-14).

Header: Run ON IPFire. Calls all per-phase validation scripts. SSHes to supportTAK-server for Phase 5. Exits 0 if all phases pass, 1 if any fail.

**Architecture (per research Pattern 6):**

```
REPO="/root/firewall-repo"
SCRIPTS="$REPO/scripts"
```

Define a `run_phase()` function that:
1. Prints a phase header banner (========== PHASE N VALIDATION ==========)
2. Runs the script (local or remote)
3. Captures exit code
4. Records result in PHASE_RESULTS array
5. Tracks overall failure state

**Phase execution order:**
1. Phase 1: `bash "$SCRIPTS/validate-phase1.sh"` (local)
2. Phase 2: `bash "$SCRIPTS/validate-phase2.sh"` (local)
3. Phase 3: `bash "$SCRIPTS/validate-phase3.sh"` (local)
4. Phase 4: `bash "$SCRIPTS/validate-phase4.sh"` (local)
5. Phase 5: Remote via SSH to supportTAK-server (see below)
6. Phase 6: `bash "$SCRIPTS/validate-phase6.sh"` (local)

**Phase 5 remote execution (per D-12, research Pitfall 4):**
- Pre-check SSH reachability: `ssh -o ConnectTimeout=10 -o BatchMode=yes opsadmin@192.168.1.101 'echo ok'`
- If SSH fails: mark Phase 5 as SKIP (not FAIL) — supportTAK-server may be offline
- If SSH succeeds: run validate-phase5.sh remotely:
  ```
  ssh -o ConnectTimeout=15 -o BatchMode=yes \
    opsadmin@192.168.1.101 \
    "source /opt/telemetry/telemetry/.env && sudo -E bash /opt/telemetry/scripts/validate-phase5.sh"
  ```
- Use the SSH key at `/root/.ssh/` (do not hardcode key filename — let SSH agent or default key handle it; if specific key needed, the deployment runbook will document the `-i` flag)

**Summary table (per D-14, console output only):**
After all phases complete, print:
```
==========================================
  VALIDATION SUITE SUMMARY
==========================================
  Phase 1: PASS
  Phase 2: PASS
  Phase 3: FAIL
  Phase 4: PASS
  Phase 5: SKIP (supportTAK-server unreachable)
  Phase 6: PASS
  ------------------------------------------
  Result: 4 PASS, 1 FAIL, 1 SKIP

  VALIDATION FAILED — see Phase 3 above
```

If all phases PASS (SKIPs allowed): print "ALL PHASES PASS" and exit 0.
If any phase FAIL: print "VALIDATION FAILED" and exit 1.
SKIPs do NOT count as failures.

**Additional flags:**
- `--phase N` to run only a single phase (e.g., `validate-all.sh --phase 6`)
- No `--full` passthrough needed — individual scripts handle their own flags
