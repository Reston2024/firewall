<task type="checkpoint:human-action">
  <name>Task 1: Complete NIC identification and substitute real MAC addresses</name>
  <what-needs-to-happen>
    The human must perform physical NIC identification on the IPFire hardware, document
    all 6 MAC addresses and PCIe bus IDs in docs/nic-map.md, and then substitute the real
    MAC addresses into all three files that contain FILL_IN_FROM_NIC_MAP placeholders.
    The executor then commits the updated files.
  </what-needs-to-happen>
  <human-steps>
    1. SSH to IPFire on port 222:
       ssh root@YOUR_IPFIRE_IP -p 222

    2. Run the NIC identification commands (from docs/nic-map.md):
       ip link show
       for iface in $(ls /sys/class/net/ | grep -v lo); do
         bus=$(ethtool -i $iface 2>/dev/null | awk '/bus-info/ {print $2}')
         mac=$(cat /sys/class/net/$iface/address)
         echo "$iface  PCIe=$bus  MAC=$mac"
       done
       lspci -nn | grep '8086:125c'

    3. Identify which physical port corresponds to which interface:
       - Use LED blink via setup utility: type 'setup' → Network Configuration → select NIC → Identify
       - OR use cable-plug method: plug cable into one port at a time, run 'ip link show' to see which goes UP

    4. Fill in docs/nic-map.md — replace every FILL_IN_FROM_NIC_MAP and FILL_IN_IP_ADDRESS
       with actual values for all 6 NICs.

    5. Fill in scripts/validate-nics.sh — replace all 6 *_EXPECTED_MAC variables:
       RED_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"     (real MAC for red0)
       GREEN_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"   (real MAC for green0)
       BLUE_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"    (real MAC for blue0)
       ORANGE_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"  (real MAC for orange0)
       GREEN1_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"  (real MAC for green1)
       GREEN2_EXPECTED_MAC="aa:bb:cc:dd:ee:XX"  (real MAC for green2)

    6. Fill in configs/udev/30-persistent-network.rules — replace each FILL_IN_FROM_NIC_MAP
       with the correct MAC for that zone's NIC.

    7. Fill in configs/ethernet/settings — replace FILL_IN_FROM_NIC_MAP with real MACs,
       and replace FILL_IN_IP_ADDRESS / FILL_IN_NETMASK with your actual IP addressing scheme.

    8. Tell the executor: "MAC addresses filled in — ready to commit"

    IMPORTANT: MAC addresses must be in lowercase colon-separated format: aa:bb:cc:dd:ee:ff
    IMPORTANT: The same MAC must appear consistently in nic-map.md, validate-nics.sh,
               udev rules, and ethernet/settings for the same zone.
  </human-steps>
  <executor-action-after-human>
    After the human confirms MACs are filled in, the executor must:

    1. Read docs/nic-map.md, scripts/validate-nics.sh, configs/udev/30-persistent-network.rules,
       and configs/ethernet/settings to verify no FILL_IN_FROM_NIC_MAP placeholders remain.

    2. If any FILL_IN_FROM_NIC_MAP still present: report which file and which line, ask human to complete.

    3. If all filled in: run these acceptance checks, then commit:
       grep 'FILL_IN_FROM_NIC_MAP' docs/nic-map.md scripts/validate-nics.sh configs/udev/30-persistent-network.rules configs/ethernet/settings
       (must return no output — zero matches = all filled in)

       git add docs/nic-map.md scripts/validate-nics.sh configs/udev/30-persistent-network.rules configs/ethernet/settings
       git commit -m "chore(01-04): substitute real MAC addresses from hardware NIC identification"
  </executor-action-after-human>
  <resume-signal>Type "MAC addresses filled in" or report which file still needs updating</resume-signal>

<task type="checkpoint:human-action">
  <name>Task 2: Deploy configs to IPFire and run reboot persistence test</name>
  <what-needs-to-happen>
    The human deploys all configuration files to the IPFire appliance, runs the PLAT-04
    steps (hostname, timezone, updates), applies firewall.local, reboots to test NIC
    persistence, then runs the zone policy steps from the runbook.
    The executor verifies git state after each human-confirmed step.
  </what-needs-to-happen>
  <human-steps>
    PART A — PLAT-04: System configuration (WUI)
    1. Open IPFire WUI at https://IPFIRE_IP:444
    2. Set hostname: System > Hostname → set to your chosen hostname → Save
    3. Set timezone: System > Time Settings → select your timezone → Save
    4. Run updates: Pakfire > Core Updates → install all pending updates
    5. Verify kernel: SSH to IPFire, run: uname -r → should return 6.18.7

    PART B — Deploy udev rules (SSH, port 222)
    6. From your dev machine, copy the repo to IPFire:
       scp -r -P 222 /path/to/Firewall root@IPFIRE_IP:/root/firewall-repo
       (or use git clone if git remote is configured)

    7. Install udev rules:
       ssh root@IPFIRE_IP -p 222
       cp /root/firewall-repo/configs/udev/30-persistent-network.rules /etc/udev/rules.d/
       udevadm control --reload-rules && udevadm trigger
       ip link show   (verify zone names appear: red0, green0, blue0, orange0, green1, green2)

    8. Install firewall.local:
       cp /root/firewall-repo/configs/firewall/firewall.local /etc/sysconfig/
       chmod +x /etc/sysconfig/firewall.local
       /etc/init.d/firewall restart
       iptables -L CUSTOMINPUT -n -v | grep -E '(222|444)'
       (both ports must show ACCEPT rules)

    9. Install backup include list:
       cp /root/firewall-repo/configs/firewall/backup-include.user /var/ipfire/backup/include.user
       backupctrl list | grep -E '(udev|firewall.local)'
       (both paths must appear)

    PART C — Reboot persistence test (CRITICAL)
    10. Reboot IPFire:
        reboot
    11. Wait 60-90 seconds for boot to complete
    12. SSH back on port 222: ssh root@IPFIRE_IP -p 222
    13. Run NIC validation:
        bash /root/firewall-repo/scripts/validate-nics.sh
        Expected: ALL NICS PASS
    14. Verify CUSTOMINPUT rules survived reboot:
        iptables -L CUSTOMINPUT -n -v | grep -E '(222|444)'
        Both ports must still show ACCEPT rules.

    PART D — Zone policies (WUI — follow docs/zone-policy-runbook.md)
    15. Enable ORANGE masquerade: WUI > Firewall > Masquerade → enable ORANGE
    16. Enable BLUE masquerade: WUI > Firewall > Masquerade → enable BLUE
    17. Block GREEN-to-ORANGE: WUI > Firewall > Firewall Rules → add DROP rule GREEN→ORANGE
    18. Block GREEN-to-BLUE: WUI > Firewall > Firewall Rules → add DROP rule GREEN→BLUE (recommended)
    19. Enable drop logging: WUI > Firewall > Firewall Options → enable Log dropped packets

    PART E — Run full validation suite (IPFire SSH)
    20. bash /root/firewall-repo/scripts/validate-phase1.sh
        Expected: ALL CHECKS PASS
    21. bash /root/firewall-repo/scripts/validate-firewall.sh
        Expected: FIREWALL CHECKS PASS

    Tell the executor: "Deployment complete" and report any failures from the validation scripts.
  </human-steps>
  <executor-action-after-human>
    After human reports deployment complete:

    1. Ask for the output of:
       bash /root/firewall-repo/scripts/validate-phase1.sh
       bash /root/firewall-repo/scripts/validate-firewall.sh

    2. If any FAIL lines: document them in the summary, flag for gap closure.

    3. If all pass: update docs/deployment-checklist.md to mark all checkboxes as done.

  </executor-action-after-human>
  <resume-signal>Type "Deployment complete — validate-phase1.sh output: [paste output]" or describe any failures</resume-signal>
