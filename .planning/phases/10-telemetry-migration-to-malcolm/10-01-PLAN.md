**Pre-flight checks (MUST run first — resolve unknowns from research):**
```bash
# 1. Check if Logstash :5044 is host-exposed
ss -tlnp | grep 5044
# 2. Check Malcolm Filebeat container ports

# 4. Check if SCP cron already exists

# 5. Check if eve.json is already being pulled
ls -la /var/log/ipfire-eve/ 2>/dev/null
# 6. Check Malcolm internal Filebeat version for version matching
docker exec malcolm-filebeat-1 filebeat version
```

**Step A: Expose Malcolm Logstash :5044 to host (if not already exposed)**
If pre-flight #1 shows port 5044 is NOT listening on host:
1. Edit `/opt/malcolm/docker-compose.yml` — find the `logstash` service section
2. Add `"5044:5044"` under its `ports:` array (if not present)
3. Restart only logstash: `cd /opt/malcolm && docker compose up -d --no-deps logstash`
4. Verify: `ss -tlnp | grep 5044` now shows a listener

**Step B: Configure Malcolm Filebeat syslog listener on UDP :5514**
1. Edit `/opt/malcolm/config/filebeat.env` — set these exact values:
   ```
   FILEBEAT_SYSLOG_UDP_LISTEN=true
   FILEBEAT_SYSLOG_UDP_PORT=5514
   FILEBEAT_SYSLOG_UDP_FORMAT=auto
   FILEBEAT_SYSLOG_TCP_LISTEN=false
   ```
2. Verify the filebeat service in `/opt/malcolm/docker-compose.yml` exposes port 5514/udp on the host. If not, add `"5514:5514/udp"` under filebeat service ports.
3. Restart filebeat: `cd /opt/malcolm && docker compose up -d --no-deps filebeat`
4. Verify: `ss -ulnp | grep 5514` shows a listener

**Step C: Configure rsyslog relay to forward to Malcolm :5514**
1. Create `/etc/rsyslog.d/20-malcolm-forward.conf` on supportTAK-server with this exact content:
   ```
   # Forward all received syslog to Malcolm Filebeat syslog listener
   *.* action(type="omfwd"
              target="127.0.0.1"
              port="5514"
              protocol="udp"
              action.resumeRetryCount="3"
              queue.type="LinkedList"
              queue.size="10000")
   ```
2. Restart rsyslog: `sudo systemctl restart rsyslog`
3. Verify: `sudo systemctl status rsyslog` shows active

**Step D: Fix IPFire syslog target IP (if pointing to old .101 address)**
If pre-flight #3 shows `192.168.1.101` instead of `192.168.1.22`:
This is a checkpoint:human-action — the user must update IPFire WUI > Logs > Log Settings > Syslog Server to `192.168.1.22` and click Save. Then restart syslog on IPFire: `ssh -i /home/opsadmin/.ssh/ipfire_ed25519 root@192.168.1.1 '/etc/init.d/syslog restart'`

If already pointing to 192.168.1.22 — skip this step.

**Step E: Install system Filebeat on supportTAK-server**
Match the version from pre-flight #6 (Malcolm's internal Filebeat version). Install:
```bash
curl -fsSL https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo gpg --dearmor -o /usr/share/keyrings/elastic.gpg
echo "deb [signed-by=/usr/share/keyrings/elastic.gpg] https://artifacts.elastic.co/packages/8.x/apt stable main" | sudo tee /etc/apt/sources.list.d/elastic-8.x.list
sudo apt update && sudo apt install filebeat
sudo systemctl enable filebeat
```

**Step F: Deploy Filebeat config for EVE JSON**
Write `/etc/filebeat/filebeat.yml` on supportTAK-server with this exact content:
```yaml
filebeat.modules:
  - module: suricata
    eve:
      enabled: true
      var.paths:
        - /var/log/ipfire-eve/eve.json

setup.ilm.enabled: false
setup.template.enabled: false

output.logstash:
  hosts: ["localhost:5044"]

logging.level: info
logging.to_files: true
logging.files:
  path: /var/log/filebeat
  name: filebeat
  keepfiles: 7
```
Also commit this file to the local repo at `telemetry-malcolm/filebeat/filebeat.yml`.

Start Filebeat: `sudo systemctl start filebeat`

**Step G: Ensure SCP cron is pulling EVE JSON**
If pre-flight #4 shows NO existing cron:
1. Create `/opt/telemetry/scripts/scp-eve.sh` on supportTAK-server:
   ```bash
   #!/bin/bash
   set -e
   SRC="root@192.168.1.1:/var/log/suricata/eve.json"
   DEST="/var/log/ipfire-eve/eve.json"
   TMP="${DEST}.tmp"
   SSH_KEY="/home/opsadmin/.ssh/ipfire_ed25519"
   scp -q -i "${SSH_KEY}" -o StrictHostKeyChecking=no "${SRC}" "${TMP}" && mv "${TMP}" "${DEST}"
   ```
2. `chmod +x /opt/telemetry/scripts/scp-eve.sh`
3. Add cron: `(crontab -l 2>/dev/null; echo '* * * * * /opt/telemetry/scripts/scp-eve.sh >> /var/log/scp-eve.log 2>&1') | crontab -`

If cron already exists and is working (eve.json recent in pre-flight #5) — leave it in place.

Also commit the SCP script to the local repo at `telemetry-malcolm/filebeat/scp-eve.sh`.

**Step H: Commit rsyslog config to repo**
Write the rsyslog forward config to `telemetry-malcolm/rsyslog/20-malcolm-forward.conf` in the local repo with the same content as Step C.
