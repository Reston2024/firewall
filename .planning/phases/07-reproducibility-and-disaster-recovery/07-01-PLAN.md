**Script structure:**

```
#!/bin/bash
# check-drift.sh — Full manifest drift detection
# Deploy to: /root/firewall-repo/scripts/check-drift.sh on IPFire
# Usage:
#   bash check-drift.sh                  Verify live files against manifest (default)
#   bash check-drift.sh --generate       Generate manifest from live files (run after deploy)
#
# Exit codes: 0=no drift, 1=error (missing manifest), 2=drift detected
#
# Distinction from check-integrity.sh (per D-15):
#   check-integrity.sh — 8 critical files, baseline stored ON IPFire, monitors for Core Update overwrites
#   check-drift.sh    — ALL managed files, manifest stored IN REPO, detects any deviation from known-good state
```

**Verify mode (default):**
- Set `MANIFEST="/root/firewall-repo/manifests/file-manifest.sha256"`
- If manifest missing, print "FAIL: Manifest not found" and exit 1
- Read each line: extract hash and file path (sha256sum output format: `hash  filepath`)
- For each entry: if file missing print "MISSING: $FILE_PATH" and increment MISSING counter; if hash matches print "OK: $FILE_PATH"; if hash differs print "CHANGED: $FILE_PATH" and increment CHANGED counter
- Print summary line: "Results: N ok, N changed, N missing"
- Exit 0 if CHANGED=0 and MISSING=0; exit 2 otherwise

**Generate mode (--generate):**
- Define MANAGED_FILES array with ALL managed config files (list below)
- For each file: if exists, run `sha256sum "$FILE"` and append to output; if missing, print warning
- Write output to `/root/firewall-repo/manifests/file-manifest.sha256`
- Print count of files baselined
- Exit 0

**Use pass()/fail()/warn() functions matching check-integrity.sh pattern.**

**MANAGED_FILES array — the full list of files the repo deploys and manages:**
```bash
MANAGED_FILES=(
  # Phase 1: Platform Foundation
  /etc/udev/rules.d/30-persistent-network.rules
  /etc/sysconfig/firewall.local
  /var/ipfire/backup/include.user
  # Phase 2: Core Network Services (repo-deployed configs only)
  /var/ipfire/dhcp/dhcpd.conf.local

  /etc/ssh/sshd_config

  /etc/suricata/suricata.yaml
  # Phase 5: Telemetry
  /etc/syslog.conf
  # Phase 6: System Hardening
  /etc/sysctl.conf
  # Integrity baseline (Phase 6)
  /root/integrity-baseline.sha256
  # Pakfire manifest (Phase 6)
  /root/firewall-repo/manifests/pakfire-manifest.txt
  # IPFire ethernet settings (WUI-managed but tracked in integrity baseline)
  /var/ipfire/ethernet/settings
)
```
