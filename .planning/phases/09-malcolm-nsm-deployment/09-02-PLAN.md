**CRITICAL: This must be done within 30 minutes of Malcolm's first start (Plan 01). If more than 30 minutes have elapsed, check if any network-* indices already exist and apply the policy retroactively.**

**Step 1: Verify OpenSearch is healthy**
```bash
sudo docker exec malcolm-opensearch-1 curl -sk http://localhost:9200/_cluster/health?pretty
```
Expect: `"status": "green"` or `"status": "yellow"`. If "red", STOP — OpenSearch is not ready.

**Step 2: Create the ISM policy**

Create the policy via the ISM API. Use `docker exec` to run curl inside the OpenSearch container (avoids TLS and auth complexity):

```bash
sudo docker exec malcolm-opensearch-1 curl -sk -XPUT \
  "http://localhost:9200/_plugins/_ism/policies/malcolm-retention" \
  -H 'Content-Type: application/json' \
  -d '{
  "policy": {
    "description": "Malcolm 30-day index retention — auto-delete indices older than 30 days",
    "default_state": "hot",
    "states": [
      {
        "name": "hot",
        "actions": [],
        "transitions": [
          {
            "state_name": "delete",
            "conditions": { "min_index_age": "30d" }
          }
        ]
      },
      {
        "name": "delete",
        "actions": [
          { "delete": {} }
        ],
        "transitions": []
      }
    ],
    "ism_template": [
      {
        "index_patterns": ["network-*", "arkime_sessions3-*"],
        "priority": 100
      }
    ]
  }
}'
```

Expect response containing `"_id": "malcolm-retention"` and `"_version": 1`.

**Step 3: Verify the policy was created**
```bash
sudo docker exec malcolm-opensearch-1 curl -sk \
  "http://localhost:9200/_plugins/_ism/policies/malcolm-retention" | python3 -m json.tool
```
Expect: JSON output with the policy definition containing `min_index_age: 30d`.

**Step 4: Check if any existing indices need the policy attached**
```bash
sudo docker exec malcolm-opensearch-1 curl -sk \
  "http://localhost:9200/_cat/indices/network-*?v"
```
If any indices exist, attach the policy retroactively:
```bash
sudo docker exec malcolm-opensearch-1 curl -sk -XPOST \
  "http://localhost:9200/_plugins/_ism/add/network-*" \
  -H 'Content-Type: application/json' \
  -d '{"policy_id": "malcolm-retention"}'
```

**Step 5: Verify ISM is managing indices**
```bash
sudo docker exec malcolm-opensearch-1 curl -sk \
  "http://localhost:9200/_plugins/_ism/explain/network-*?pretty" 2>/dev/null || echo "No network-* indices yet (expected if Malcolm just started with no data)"
```
If indices exist, expect `"policy_id": "malcolm-retention"` in the output. If no indices yet, this is expected — the ISM template will auto-attach when they are created.
