   - Use chromadb.PersistentClient(path="/var/lib/chromadb") — NEVER chromadb.Client()
   - Use collection name "firewall-corpus" with metadata {"hnsw:space": "cosine"}
   - Use HuggingFaceEmbeddings with model_name="all-MiniLM-L6-v2" and model_kwargs={"device": "cpu"}
   - Implement two-stage chunking: MarkdownHeaderTextSplitter (split on #, ##, ###, strip_headers=False) THEN RecursiveCharacterTextSplitter (chunk_size=1800, chunk_overlap=200)
   - Route .sh/.py files through RecursiveCharacterTextSplitter only (skip header splitter)
   - Prepend breadcrumb to every chunk: "[source_path] / # h1 / ## h2 / ### h3"
   - Index these corpus globs from repo root:
     - decisions/*.md (15 ADR files)
     - docs/*.md (~8 runbook/guide files)
     - docs/benchmarks/*.md (1 benchmark file)
     - contracts/*.md and contracts/*.json (2 files)
     - .planning/REQUIREMENTS.md
     - .planning/research/SUMMARY.md
     - .planning/research/STACK.md
     - scripts/validate-*.sh (~12 validation scripts)
   - Print per-file chunk count and total chunk count
   - Verify persistence by re-opening PersistentClient and checking collection count
   - Set HF_HUB_OFFLINE=1 environment variable before embedding to prevent network calls during indexing

6. Create scripts/rag_query.py based on the skeleton in 12-RESEARCH.md (Pattern: Query Interface Skeleton). The script MUST:
   - Accept a query string as CLI argument
   - Default k=3 (NOT k=5 — OLLAMA_CONTEXT_LENGTH=2048 constraint, per research Pitfall 6)
   - Accept optional --k flag to override
   - Use same embedding model (all-MiniLM-L6-v2) and persist_directory (/var/lib/chromadb)
   - Print each chunk with score, source, and first 500 chars of content
   - Export query_corpus() function for import by Phase 13 triage worker

