  <action>
All commands run via SSH to opsadmin@192.168.1.22. Use `sudo` for privileged operations.

STEP 1 — Check for existing Ollama install:
```bash
ssh opsadmin@192.168.1.22 "ollama --version 2>/dev/null || echo 'NOT_INSTALLED'"
```
If already installed, skip to Step 3. If not installed, proceed to Step 2.

STEP 2 — Install Ollama:
```bash
ssh opsadmin@192.168.1.22 "curl -fsSL https://ollama.com/install.sh | sh"
```
The installer creates a systemd service that starts immediately. The service defaults to binding 0.0.0.0:11434. We MUST fix this before pulling any model.

STEP 3 — Stop Ollama and apply security override (ADR-E02 CRITICAL):
```bash
ssh opsadmin@192.168.1.22 "sudo systemctl stop ollama"
```
Then create the override file:
```bash
ssh opsadmin@192.168.1.22 "sudo mkdir -p /etc/systemd/system/ollama.service.d && sudo tee /etc/systemd/system/ollama.service.d/override.conf > /dev/null << 'OVERRIDE'
[Service]
Environment=\"OLLAMA_HOST=127.0.0.1\"
Environment=\"OLLAMA_KEEP_ALIVE=5m\"
OVERRIDE"
```
Reload and restart:
```bash
ssh opsadmin@192.168.1.22 "sudo systemctl daemon-reload && sudo systemctl restart ollama"
```

STEP 4 — Verify security binding BEFORE proceeding (ADR-E02 gate):
```bash
ssh opsadmin@192.168.1.22 "ss -tlnp | grep 11434"
```
Output MUST show `127.0.0.1:11434`. If it shows `0.0.0.0:11434`, STOP and debug the override file. Do NOT proceed to model pull until binding is confirmed.

STEP 5 — Apply ufw defense-in-depth (ADR-E02):
```bash
ssh opsadmin@192.168.1.22 "sudo ufw deny 11434 && sudo ufw status | grep 11434"
```
Expected: `11434 DENY Anywhere`

STEP 6 — Pull Foundation-Sec-8B Q4_K_M from HuggingFace:
```bash
ssh opsadmin@192.168.1.22 "ollama pull hf.co/fdtn-ai/Foundation-Sec-8B-Q4_K_M-GGUF"
```
This downloads ~4.92 GB. May take 5-15 minutes depending on network speed. Do NOT use `fdtn-ai/Foundation-Sec-8B-Q4_K_M` (not in Ollama registry). The `hf.co/` prefix is mandatory.

STEP 7 — Verify model responds to a security question:
```bash
ssh opsadmin@192.168.1.22 "ollama run hf.co/fdtn-ai/Foundation-Sec-8B-Q4_K_M-GGUF 'What is a SQL injection attack and how do you mitigate it? Be concise.' 2>&1 | head -20"
```
Confirm the model produces a coherent security-relevant response.

STEP 8 — Verify model appears in ollama list:
```bash
ssh opsadmin@192.168.1.22 "ollama list"
```
Should show `hf.co/fdtn-ai/Foundation-Sec-8B-Q4_K_M-GGUF` with size ~4.9 GB.
