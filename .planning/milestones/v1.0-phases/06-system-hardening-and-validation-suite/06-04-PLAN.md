  <how-to-verify>
    1. On IPFire, reboot the system:
       ```
       reboot
       ```
    2. Wait for IPFire to come back online (typically 1-2 minutes)
    3. SSH back into IPFire and run the reboot comparison:
       ```
       bash /root/firewall-repo/scripts/validate-reboot.sh --compare
       ```
    4. Expected: "PASS: System state identical after reboot" — no differences in sysctl values, listening services, iptables rules, or config file hashes
    5. If differences found: review the diff output. Common expected differences:
       - Timestamps will differ (this is normal — the script compares values, not timestamps)
       - If sysctl values reverted: check that /etc/sysctl.conf was properly saved
       - If services differ: check /etc/rc.d/rc3.d/ symlinks
  </how-to-verify>
  <verify>validate-reboot.sh --compare exits 0</verify>
  <done>No state differences after reboot — sysctl, iptables, services, and config hashes all match pre-reboot snapshot</done>
