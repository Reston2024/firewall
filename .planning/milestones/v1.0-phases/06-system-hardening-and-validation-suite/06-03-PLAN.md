<task type="auto">
  <name>Task 1: Create hardening deployment runbook</name>
  <files>docs/hardening-deployment-runbook.md</files>
  <action>
Create `docs/hardening-deployment-runbook.md` with numbered steps for deploying all Phase 6 hardening to IPFire. Follow the same format as existing runbooks (telemetry-deployment-runbook.md, suricata-ids-runbook.md).

**Sections:**

**Section 0: Prerequisites**
- All Plan 01 and Plan 02 scripts committed to repo
- SSH access to IPFire as root
- SSH access from IPFire to supportTAK-server (for validate-all.sh Phase 5)

**Section 1: Deploy scripts to IPFire**
```
# From dev machine — sync entire repo to IPFire
scp -r scripts/ root@IPFIRE_IP:/root/firewall-repo/scripts/
scp -r configs/ root@IPFIRE_IP:/root/firewall-repo/configs/
scp -r manifests/ root@IPFIRE_IP:/root/firewall-repo/manifests/
scp -r docs/ root@IPFIRE_IP:/root/firewall-repo/docs/
```

**Section 2: Apply sysctl hardening (HARD-04)**
```
# On IPFire — read existing sysctl.conf before modifying
cat /etc/sysctl.conf
# Append hardening params (DO NOT overwrite)
cat /root/firewall-repo/configs/hardening/sysctl-hardening.conf >> /etc/sysctl.conf
# Apply immediately
sysctl -p /etc/sysctl.conf
# Verify critical params
sysctl net.ipv4.conf.all.send_redirects   # must be 0
sysctl net.ipv4.ip_forward                # MUST be 1
```

**Section 3: File permission lockdown (HARD-02)**
```
# On IPFire
chmod 600 /etc/ssh/sshd_config
chmod 700 /etc/sysconfig/firewall.local
chmod 700 /root/.ssh/
chmod 600 /root/.ssh/authorized_keys 2>/dev/null
```

**Section 4: Service audit (HARD-01)**
```
# On IPFire — enumerate listening services
ss -tlnp
# Expected: unbound:53, sshd:22, httpd:81/444/1013
# Any other listener is a finding — investigate before proceeding

# Generate Pakfire manifest
ls /opt/pakfire/db/installed/ > /root/firewall-repo/manifests/pakfire-manifest.txt
# Compare against expected
diff /root/firewall-repo/manifests/pakfire-manifest-expected.txt /root/firewall-repo/manifests/pakfire-manifest.txt
```
Note: If unexpected services found, document in decision-log/ and either disable (mv symlink from /etc/rc.d/rc3.d/) or justify keeping.

**Section 5: Deploy backup include list**
```
# On IPFire
cp /root/firewall-repo/configs/firewall/backup-include.user /var/ipfire/backup/include.user
```

**Section 6: WUI certificate documentation (HARD-05)**
```
# On IPFire — extract certificate details
openssl x509 -in /etc/httpd/server.crt -noout -subject -issuer -dates -fingerprint -sha256
openssl x509 -in /etc/httpd/server-ecdsa.crt -noout -subject -issuer -dates -fingerprint -sha256
# Record output — update docs/wui-certificate.md with actual values
```

**Section 7: Create integrity baseline (HARD-03)**
```
# On IPFire — AFTER all hardening is applied
bash /root/firewall-repo/scripts/check-integrity.sh --create-baseline
# Lock down baseline file
chmod 600 /root/integrity-baseline.sha256
# Verify
bash /root/firewall-repo/scripts/check-integrity.sh --verify
```
IMPORTANT: Create baseline AFTER sysctl and permission changes, not before.

**Section 8: Run validate-phase6.sh**
```
bash /root/firewall-repo/scripts/validate-phase6.sh
# Expected: all HARD-01 through HARD-05 checks pass
```

**Section 9: Capture pre-reboot snapshot (VAL-08 prep)**
```
bash /root/firewall-repo/scripts/validate-reboot.sh --snapshot
# Output: /root/reboot-snapshot.txt
# DO NOT reboot yet — Plan 04 handles reboot persistence test
```

Include notes about the order constraint: sysctl first, then permissions, then integrity baseline LAST (since baseline hashes the files after changes).
